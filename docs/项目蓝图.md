## 二、项目总目标

官网后端的目标，是把企业官网从静态展示升级为可配置、可缓存、可审计、可持续迭代的后台内容系统，形成 Admin 管理端与 Portal 公共端两条稳定链路。

### 目标能力

- 后台可维护全站内容、媒体资源、线索数据与审计记录。
- 前台只读取可公开展示数据，支持缓存与多实例部署。
- 所有写操作可追踪、可回滚、可校验并发冲突。
- 全站内容模块按统一的可见性、逻辑删除、排序、媒体绑定和缓存失效规则实现。

### 当前边界

- 当前架构为单体 Spring Boot 应用，可多实例部署。
- 当前后台权限模型默认只有 `ADMINISTRATOR`。
- 当前媒体存储默认走本地文件存储与公共访问路径。
- 当前默认生命周期只包含显示、隐藏、逻辑删除，不引入草稿、审批和定时发布。

### 模块实施原则

- 先完成公共底座，再按门户展示优先级推进内容模块。
- 所有内容模块优先复用同一套排序、可见性、逻辑删除和媒体生命周期规则。
- 新模块进入实现前，必须先在本节补充其模块归属和核心依赖。

## 五、技术实现蓝图

### 1. 分层原则

- Controller 负责接参、校验、鉴权入口与统一响应，不直接访问 Mapper。
- Application Service 负责跨模块编排与只读聚合。
- Module Service 负责事务、校验、审计、媒体绑定和缓存失效注册。
- Mapper 只负责数据访问。
- Infrastructure 负责缓存、存储、安全适配和审计支撑。

### 2. 一致性原则

- 写操作在事务内完成业务数据、审计记录和媒体引用维护。
- 缓存失效在事务提交后触发，并保留延迟二次删除兜底。
- 逻辑删除统一使用 `deleted_marker`，活跃值为 `0`。
- 唯一业务键通过数据库联合唯一索引兜底，避免只靠 Service 层判重。

### 3. 安全原则

- Admin 写接口必须登录、校验角色并通过 CSRF 校验。
- Portal 接口不得返回隐藏、删除或后台审计字段。
- 线索、导出、敏感信息查看和高风险写操作必须有审计记录。
- 文件上传必须经过类型、大小、路径与内容校验。

## 六、实施阶段与里程碑

| 阶段 | 目标能力 | 前置依赖 | 完成判定 |
|---|---|---|---|
| 第一阶段：基础底座 | 登录鉴权、统一响应、错误码、媒体中心、审计、缓存基线、Docker/Flyway 基线 | 无 | 应用可启动，具备统一写入规范与验证环境 |
| 第二阶段：官网核心内容 | 站点配置、导航、首页 Banner、首页数据卡片、荣誉、客户 Logo | 第一阶段 | 官网主干页面内容可由后台动态维护 |
| 第三阶段：内容扩展 | AI 战略、核心能力、产品矩阵、行业方案、案例、关于我们 | 第一、二阶段 | 官网主要展示内容全部进入后台管理 |
| 第四阶段：线索闭环 | 联系信息、合作方向、Portal 留资、后台线索看板、导出、限流 | 第一阶段 | 官网形成内容展示到业务留资的完整闭环 |

### 里程碑控制要求

- 每个阶段完成后，必须同步补齐接口文档、必要测试和缓存策略说明。
- 阶段内新增公共机制，必须先更新架构文档或规则文档，再扩散到业务模块。
- 未完成底座能力前，不应跳过统一规则直接堆叠业务页面接口。

## 七、质量门禁与验收标准

- Admin 写接口必须鉴权，且仅允许管理员角色。
- Portal 接口只能返回 `visible=1` 且 `deleted_marker=0` 的数据。
- 内容变更必须写审计。
- 引用媒体必须校验存在性，并处理绑定与解绑。
- 高并发编辑必须使用乐观锁。
- Portal 缓存必须在事务提交后失效，并具备延迟二次删除。
- 所有新增或变更 API 必须同步更新 [接口文档](接口文档.md)。
- 关键业务必须能在 Docker 环境下完成验证。
- 新表、新索引和逻辑删除策略必须符合 [数据库设计说明](数据库设计说明.md)。
- 新功能上线前，必须能在本蓝图中定位其模块、对象、阶段和依赖关系。

## 八、风险与待决策项

| 项目 | 当前默认方案 | 触发重审条件 |
|---|---|---|
| 图片/视频/PDF 存储 | 先复用现有本地媒体存储 | 引入对象存储或多地域部署 |
| 后台权限模型 | 仅 `ADMINISTRATOR` | 引入细粒度 RBAC |
| Portal 缓存 | Redis 或等价共享缓存 | 多实例一致性策略调整 |
| 数据初始化 | 仅初始化必要默认数据 | 需要更完整种子内容 |
| 线索安全 | 默认脱敏，管理员可查看 | 新增更高敏感字段或导出要求 |
| 大文件处理 | 默认不引入独立异步媒体服务 | 视频、PDF 或批量处理显著增加 |

- 新增模块时，先在本蓝图登记模块域、核心对象、依赖和阶段，再进入实现。
